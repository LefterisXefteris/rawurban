<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| product card client -> `/api/cart` | Untrusted click events cross from the browser into cart mutation requests |
| newsletter form input -> client UI state | User-entered email text crosses into a rendered success flow |
| footer links -> Next.js router | Click targets navigate users into internal legal routes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `components/cards/ProductCard.tsx` | mitigate | Derive `merchandiseId` only from server-fetched variant data in `lib/index.ts`; never accept arbitrary ids from DOM attributes or query params; keep the CTA as a `button type="button"` separate from the product `Link`. |
| T-02-02 | I | `components/footer/NewsletterSignup.tsx` | mitigate | Render only static success copy after submit; call `preventDefault()` and do not interpolate the raw email value back into JSX. |
| T-02-03 | E | `app/page.tsx` footer legal links | mitigate | Use hardcoded internal `Link` hrefs (`/privacy-policy`, `/terms-of-service`) only; do not construct destinations from user input or external URLs. |
| T-02-04 | D | `components/cards/ProductCard.tsx` | accept | Quick Add can be clicked repeatedly, but the component already serializes requests through a local loading state and the existing cart context; no high-risk resource amplification is introduced in ASVS L1 scope. |
</threat_model>
